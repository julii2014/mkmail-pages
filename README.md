# mkmail-pages

## 项目简介

`mkmail-pages` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 页面的存放与展示空间。

通过 GitHub Pages 或其他静态托管服务，您可以方便地访问这些页面内容。页面内容可能包括信息展示、工具页面、临时存档等，具体用途随归档内容而定。

## 目录说明

- `pages/`：存放所有归档的 HTML 页面文件，按需组织子目录或直接存放。
- `assets/`（可选）：存放页面所需的公共资源，如样式表、脚本、图片等。
- `index.html`：仓库根目录下的首页，提供页面列表或导航入口（如有）。

## 页面归档说明

- 每个 HTML 页面应为独立、自包含的文档，尽量减少对外部资源的依赖。
- 页面文件名建议使用有意义的英文或数字标识，避免特殊字符。
- 如需引用公共资源，请统一放置在 `assets/` 目录下，并在页面中正确引用相对路径。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面内容。
- 不承诺对任何页面的长期可用性或内容准确性。
- 如果您发现页面存在问题或有改进建议，欢迎提交 Issue 或 Pull Request。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在 `pages/` 目录下添加新的 HTML 文件，或修改现有文件。
3. 将更新推送到 GitHub 仓库，并确保 GitHub Pages 已启用（如需要）。
4. 通过仓库设置的 Pages 链接访问对应页面。
